## Codebase Patterns

### Exact text mutation helper
- **Where**: `scripts/apply-worker.ts:278-299`
- **Pattern**: Use `Bun.file(...).text()` + literal `includes`/`replace` so apply-worker mutations only touch the first exact string match, and return a typed `string_mismatch` result instead of throwing when the proposal no longer matches disk.
- **Example**: `if (!original.includes(before)) { return { file: filePath, status: "string_mismatch", before, after }; }`

### Decision-ordered proposal parsing
- **Where**: `scripts/apply-worker.ts:142-275`
- **Pattern**: Parse all H3 proposal sections first, index them by proposal number, then return issues in `decision.json.selected_issues` order so downstream apply steps follow the orchestrator's selected-issue order even when proposal headings include composite severities like `[CRITICAL+HIGH]`.
- **Example**: `const sectionsByIndex = new Map(sections.map((section) => [section.index, section]));`

### Mutation file heuristics
- **Where**: `scripts/apply-worker.ts:171-230`
- **Pattern**: Extract file paths only from backticked target-file entries that actually look like paths, then resolve ambiguous multi-file before/after pairs by scoring change labels against file names and falling back to positional order.
- **Example**: `return filesTouched[pairIndex] ?? filesTouched[0] ?? "";`
